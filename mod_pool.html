ProFTPD module mod_pool


The mod_pool module is used to periodically log the status of the memory pools used by a ProFTPD session.

Installation instructions are discussed here.

The most current version of mod_pool can be found at: 

  https://github.com/Castaglia/proftpd-mod_pool

Directives

  • PoolEngine
  • PoolEvents
  • PoolLogs

PoolEngine

Syntax: PoolEngine on|off
Default: off
Context: server config, <VirtualHost>, <Global>
Module: mod_pool
Compatibility: 1.3.6rc2 and later

The PoolEngine directive enables or disables the mod_pool module.

PoolEvents

Syntax: PoolEvents event1 ...
Default: PoolEvents ALL
Context: server config, <VirtualHost>, <Global>
Module: mod_pool
Compatibility: 1.3.6rc2 and later

The PoolEvents directive configures which for which events the mod_pool module will log the memory pool usage.

The supported events are:

  • ALL
  • Directories
  • Downloads
  • Misc
  • Sessions
  • Transfers
  • Uploads

Installation

To install mod_pool, copy the mod_pool.c file into the third-party module area in the proftpd source code: 
  $ cp mod_pool.c proftpd-dir/contrib/
after unpacking the latest proftpd-1.3.x source code. For including mod_pool as a staticly linked module: 
  $ ./configure --with-modules=mod_pool ...
Alternatively, mod_pool can be built as a DSO module: 
  $ ./configure --enable-dso --with-shared=mod_pool ...
Then follow the usual steps: 
  $ make
  $ make install
